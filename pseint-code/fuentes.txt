 Inicio Rapido===============Si solo quiere compilar y utilizar el programa instale las dependencias (ver requisitos) y utilice el comando "ming32-make win32" si esta en Microsoft Windows, o el comando"make linux" si esta en GNU/Linux o Mac Os. Los binarios se ubicarán en el directorio bin. Desde allí puede ejecutar wxPSeInt para comenzar a utilizar el programa. Requisitos para compilar PSeInt=================================Para compilar pseint se requieren las bibliotecas wxWidgets-2.8, OpenGL y glut. En la mayoría de los sistemas GNU/Linux estas se pueden conseguir facilmente a través del repositorio de la distribución (aunque usualmente la versión disponible de wx es Unicode mientras que la que utilizo en mis compilaciones es ANSI, se requiere editar las variables LIBS Y CFLAGS en el archivo wxPSeInt/Makefile.lnx para utilizar la version Unicode). Si utiliza Windows puede descargar el IDE ZinjaI que incluye el compilador gcc y las bibliotecas necesarias. También puede encontrar instrucciones más detalladas de cómo descargar y compilar las últimas versiones en http://cucarachasracing.blogspot.com.ar/2012/11/compilar-la-version-del-repositorio-en.html (linux)http://cucarachasracing.blogspot.com.ar/2012/11/compilar-la-version-del-repositorio-en_4.html (windows) Acerca de los fuentes de PSeInt=================================Contenido de los directorios:  pseint: fuentes del interprete (verifica sintaxis e interpreta un algoritmo)  psdraw3: fuentes del editor de diagramas de flujo (permite editar el diagrama)  psdrawE: fuentes del exportador de diagramas de flujo (genera imagenes png/jpg/bmp)  psexport: fuentes del exportador (convierte de pseudocodigo a codigo C++)  wxPSeInt: fuentes de la nueva interfaz (editor de pseudocódigo, e interfaz principal)  psterm: fuentes de la terminal que se utiliza para ejecutar desde wxPSeInt  updatem: fuentes del programa que busca actualizaciones  test: pseudocódigos con casos de prueba y scripts para correrlos de forma automática  bin: demas archivos necesarios para ejecutar wxPSeInt (imagenes, documentacion, ayuda, etc).  dist: archivos adicionales para generar paquetes e instaladoresPuede encontrar más información de la función de cada módulo y de cómo se comunican entreellos en http://cucarachasracing.blogspot.com.ar/2012/12/destripando-pseint.htmlAdentro de cada directorio hay tres Makefiles:  Makefile.lnx para compilar las versiones para GNU/Linux  Makefile.w32 para compilar las versiones para Windows desde Windows, con MinGW  Makefile.wine para compilar las versiones para Windows desde GNU/Linux, con Wine+MinGW  Makefile.m32 para compilar las versiones para Mac OS X i686 desde GNU/Linux, con IMCROSS  Makefile.ppc para compilar las versiones para Mac OS X powerpc desde GNU/Linux, con IMCROSSPara compilar en o para Windows puede que necesite retocar las variables definidas en mingw.conf. Los archivos intermedios de la compilación se ubican en temp.Además, hay un archivo Makefile.pack que sirve para generar los paquetes e instaladoresde todo el software (solo desde GNU/Linux, y requiere wine+mingw+wxwidgets+nsis).Para poder visualizar íconos en el editor de diagrama de flujo en Windows, deberá modificarel archivo psdraw2/Makefile.w32 agregando la bandera de compilación -D_USE_TEXTURES, y las opciones necesarias para utilizar la biblioteca libpng (puede tomar como ejemploel makefile para wine). El makefile por defecto no lo incluye para poder compilarsin errores con una instalación de ZinjaI estándar, ya que la biblioteca libpng (ysu dependencia zlib) deben descargarse por separado e instalarse manualmente.Los fuentes son exactamente los mismos tanto para GNU/Linux como para Windows.Los paquetes que se distribuyen desde el sitio oficial se construyen con gcc y enGNU/Linux (Ubuntu 10.10), aún la versión para de Windows (utilizando mingw, wine y Slackware 14.2).El instalador para Windows se construye con NSIS (nsis.sourceforge.net). El framework utilizado para el desarrollo de la GUI es wxWidgets (www.wxwidgets.org)El lenguaje empleado en todos los fuentes es C++.En cada directorio de fuentes hay un archivo .zpr. Este es el archivo de proyecto.Se abren con un IDE llamado ZinjaI (disponible en http://zinjai.sourceforge.net).Actualizaciones: http://pseint.sourceforge.netPara enviar comentarios, consultas, reportes de errores, etc, acceda a los foros en:http://pseint.sourceforge.net/index.php?page=contacto.phpTodo se distribuye bajo licensia GPL (ver pack/help/gpl.html).Pablo Novara (zaskar_84@yahoo.com.ar)
